# 1. 方案选型

| API | 比分 | 实时性 | 免费层 | 推荐度 |
|---|---|---|---|---|
| **football-data.org** | ✓ | 延迟 1—2 分钟 | 10 次/分钟，免费 | ★★★★ 推荐 |
| API-Football | ✓ | 实时 | 有限额，需付费生产 | ★★★★ 备选 |
| SportMonks | ✓ | 实时 | 无免费层 | ★★★ 数据质量高但贵 |

**结论**：赛果写入只需比赛结束后获取结果，1—2 分钟延迟可接受，使用 **football-data.org** 免费层即可满足需求。若后续需要实时比分展示，可切换至 API-Football。

# 2. 所需数据字段

| 本地字段 | API 对应字段 | 说明 |
|---|---|---|
| `match_id` | `id` | 赛事唯一 ID |
| `home_team` | `homeTeam.name` | 主队名称 |
| `away_team` | `awayTeam.name` | 客队名称 |
| `kickoff_time` | `utcDate` | 开赛时间，转为时间戳存储 |
| `status` | `status` | 赛事状态，见状态映射表 |
| `result` | `score.winner` | 赛果，见赛果映射表 |

## 2.1 状态映射

| API 返回值 | 本地 status |
|---|---|
| SCHEDULED / TIMED | upcoming |
| IN_PLAY / PAUSED | ongoing |
| FINISHED | finished |

## 2.2 赛果映射

| API 返回值 | 本地 result |
|---|---|
| HOME_TEAM | home |
| DRAW | draw |
| AWAY_TEAM | away |

# 3. 同步策略

## 3.1 赛程同步（队名、开赛时间）

| 频率 | 时间 | 说明 |
|---|---|---|
| 每日 1 次 | **06:00 UTC**（北京时间 14:00） | 所有场次最晚 ~03:00 UTC 结束，留 3 小时等 API 更新；距 12:00 UTC 配置生效还有 6 小时，运营有足够时间完成配置。小组赛、淘汰赛统一适用 |

管理页提供**「同步赛程」**手动按钮，供运营在临时改期或同步失败时手动触发。

## 3.2 赛果轮询（比赛进行中）

| 阶段 | 频率 | 说明 |
|---|---|---|
| 比赛日赛前 | 每小时 1 次 | 防止临时改期 |
| 比赛进行中 | 每 60 秒 1 次 | 免费层限速 10 次/分钟，2 场同时进行仍在限额内 |
| 比赛结束 | 捕到 FINISHED 即停止 | 写入 `match_schedule.result` 后不再轮询该场 |

---

# 4. 结算触发

1. 定时任务轮询到 `status = finished` 时，写入 `match_schedule.result`
2. 写入成功后触发竞猜结算逻辑，遍历该场所有 `user_prediction` 记录
3. 结算完成后更新 `world_coin_balance` 并写入 `world_coin_tx`
